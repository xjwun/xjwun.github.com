3. Streams

What’s steam?

Sequence of elements

A stream provides an interface to a sequenced set of values of a specific element type. However, streams don’t actually store elements; they’re computed on demand.

Operations

Operations such as filter, sorted, and map, which can be connected together to form a pipeline

Operations such as collect, findFirst, and allMatch, which terminate the pipeline and return a result

Filtering

  1. filter : Takes a Predicate object as an argument and returns a stream including all elements that match the predicate
  2. distinct : Returns a stream with unique elements (according to the imple‐ mentation of equals for a stream element)
  3. limit : Returns a stream that is no longer than a certain size
  4. skip : Returns a stream with the first n number of elements discarded

Matching

  1. anyMatch
  2. allMatch
  3. noneMatch

Finding

  1. findFirst
  2. findAny

Mapping

map() takes a Function object as an argument to turn the elements of a stream into another type.

Reducing

Collectors

use Collector object to accumulate elements into final result

Parallel Streams

steam API support easy data parallelism. Streams API will use the Fork/Join framework, leverage the multiple cores of your machine.
steam API support easy data parallelism. Streams API will use the Fork/Join framework, leverage the multiple cores of your machine.

Two way

  1. parallelStream()
  2. convert existing Stream into parallel STream by using parallel() method

Factors to consider if use parallel

  1. Splittability : data structure like arrays are easily splittable, but likedlist or files offer poor splittability
  2. Cost per element : more expensive, more benefit from parallelism
  3. Boxing : perfer to use primitives instead of objects if possible
  4. Size : parallel setup cost may amortized over processing of element, parallel speedup will outweigh the setup cose
  5. Number of Cores
